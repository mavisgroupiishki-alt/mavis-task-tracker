# MAVIS GROUP — простой трекинг задач

Готовый MVP-приложение для команды:

- задачи на день / неделю / месяц;
- выбор конкретного сотрудника;
- график загрузки по часам;
- статусы задач;
- просроченные задачи;
- быстрое добавление задачи;
- импорт задач из Google Таблицы через CSV;
- хранение задач в общей базе Supabase;
- деплой через GitHub + Render.

---

## 1. Как быстро запустить локально

```bash
npm install
npm run dev
```

Если Supabase ещё не подключён, приложение откроется в локальном режиме. Это удобно для проверки интерфейса, но задачи будут храниться только в браузере.

---

## 2. Настройка Supabase

1. Создайте проект в Supabase.
2. Откройте раздел **SQL Editor**.
3. Вставьте содержимое файла `supabase.sql`.
4. Нажмите **Run**.
5. Откройте **Project Settings → API**.
6. Скопируйте:
   - Project URL;
   - anon public key.

Создайте файл `.env` на основе `.env.example`:

```bash
VITE_SUPABASE_URL=https://YOUR_PROJECT_ID.supabase.co
VITE_SUPABASE_ANON_KEY=YOUR_SUPABASE_ANON_KEY
```

---

## 3. Деплой на Render

1. Создайте репозиторий на GitHub.
2. Загрузите туда все файлы проекта.
3. В Render создайте **New → Static Site**.
4. Подключите GitHub-репозиторий.
5. Укажите команды:

```bash
Build Command: npm install && npm run build
Publish Directory: dist
```

6. В Render добавьте Environment Variables:

```bash
VITE_SUPABASE_URL=ваш Supabase Project URL
VITE_SUPABASE_ANON_KEY=ваш Supabase anon public key
```

7. Нажмите **Deploy**.

После деплоя Render выдаст ссылку на приложение. Эту ссылку можно отправить сотрудникам.

## 4. Формат CSV для импорта из Google Таблицы

В Google Таблице должны быть такие колонки:

```text
Задача,Ответственный,Срок,Период,Статус,Приоритет,Часы,Измеримый результат
```

Пример строки:

```text
Проверить сделки в работе,РОП,2026-06-03,День,В работе,Высокий,2,По всем сделкам стоит следующий шаг
```

Допустимые значения:

**Период:**
- День
- Неделя
- Месяц

**Статус:**
- Новая
- В работе
- На проверке
- Готово

**Приоритет:**
- Низкий
- Средний
- Высокий

Как выгрузить:

**Google Таблица → Файл → Скачать → CSV**

Потом в приложении нажмите **Импорт CSV**.

## 5. Как поменять список сотрудников

Откройте файл:

```text
src/App.jsx
```

Найдите блок:

```js
const team = [
  'Виктория',
  'РОП',
  'Руководитель экспертного отдела',
  'Маркетинг',
  'Эксперт 1',
  'Эксперт 2',
  'Менеджер 1',
  'Менеджер 2',
];
```

Замените имена на ваших сотрудников.

## 6. Важно по безопасности

В `supabase.sql` включён быстрый MVP-режим: любой, у кого есть ссылка на приложение, может смотреть, добавлять, менять и удалять задачи.

Это сделано для максимально быстрого запуска без логинов и паролей.

Позже можно добавить:

- авторизацию по email;
- роли администратора и сотрудника;
- запрет удаления задач для обычных пользователей;
- историю изменений.
